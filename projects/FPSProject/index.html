项目定位

Commander Frontline 是一个基于 Unreal Engine 5.4 的多人 FPS / 轻策略玩法原型。项目从基础 FPS 模板扩展出回合制对抗模式，重点验证服务端权威战斗、非对称职业分工、指挥官命令、AI 兵线压力、中立目标争夺和基地防御等系统如何在多人 PIE 环境中协同运行。

我的职责

  • 设计双队回合流程、队伍/职业分配、得分、能量、复活和胜负结算逻辑
  • 实现 Commander、Assault、Engineer 三类职业定位与主动技能
  • 构建 FieldBot、Lane Minion、Neutral Guard 等 AI 单位，形成兵线压力和中立目标战斗
  • 将伤害、奖励、转化、Final Stand 和回合状态放在服务端权威路径处理
  • 使用原生 Canvas HUD 展示比分、阶段、能量、生命值、技能状态、世界血条、通知和回合结果

结果亮点

  • 形成了可运行的多人玩法闭环：准备阶段、正式回合、目标争夺、防御塔摧毁、Final Stand 和回合结算
  • C++ 代码按 GameMode、Player、AI、World、HUD、GameState、PlayerState 和 Tests 等职责拆分，便于继续扩展和审查
  • 项目文档覆盖设计说明、技术架构、多人 PIE 测试、AI 蓝图设置和动画配置，方便复现和迭代

🛠️ 技术栈

  • Engine: Unreal Engine 5.4
  • Language: C++17
  • Networking: Server Authoritative Multiplayer
  • Gameplay: Round Flow, Role Skills, AI Orders, Neutral Objectives
  • Key Modules: GameMode, GameState, PlayerState, AI, World Actors, Canvas HUD, Automation Tests

📝 项目核心技术方案

本项目将 FPS 射击操作与轻策略指挥玩法结合，核心目标是让多人对抗、职业技能、AI 兵线和目标奖励都能通过清晰的服务端状态机驱动。

1. 服务端权威的回合与战斗流程

  • 状态集中： 回合阶段、队伍分配、得分、能量、复活、目标奖励和胜负结算由服务端 GameMode 统一推进。
  • 客户端表现： 客户端通过 RPC 请求操作，并读取复制状态更新 HUD、提示和世界血条，减少本地状态分歧。

2. 非对称职业与指挥玩法

  • Commander： 负责 AI 命令、兵线策略和基地防守，防御塔被摧毁后进入 Final Stand。
  • Assault： 使用 Overdrive 获得短时间机动与输出强化，承担正面交战压力。
  • Engineer： 部署防御屏障，提供区域控制和防守支援。

3. AI 兵线与中立目标

  • 兵线压力： FieldBot、Lane Minion 和 Neutral Guard 共同制造推进、拦截和目标争夺。
  • NeutralCore 奖励： 中立事件会带来分数、团队能量、XP 和后续波次增益，让玩家在击杀之外有明确的战略目标。

4. 原生 HUD 与测试文档

  • Canvas HUD： 使用原生 Canvas 绘制比分、阶段、能量、技能状态、通知、转化面板和回合结果。
  • 复现路径： 仓库保留技术文档、多人 PIE 说明和自动化测试目录，便于在编辑器中验证复制逻辑和核心玩法状态。
